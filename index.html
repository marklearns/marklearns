01
Idea + Prompt Engineering
Finding what people actually search for
✓
▼

Every money-making website starts with demand that already exists. Don't invent demand — find it and serve it better than anyone else.

Use Google's autocomplete to find what people are searching. Type a niche + common questions: "what does my [X] say about me", "am I [adjective]", "what [adjective] person am I". These are goldmines.

PROMPT TEMPLATE: "Create an engaging, viral [TOOL TYPE] website. The tool should feel personalized and surprising. Results should be shareable. Include: - Input form (name/date/question) - Dynamic AI result display - Share button Design: dark, modern, mobile-first."
💡 Pro TipTools that make people feel "seen" get shared. Horoscopes, personality tests, name meanings — these tap into identity. Identity = shares = free traffic.
02
Claude Website Generation
Getting a full working site in one message
✓
▼

Claude (claude.ai) is the best tool for generating complete, production-ready HTML websites in a single prompt. Here's how to use it effectively.

Start a new conversation. Paste your prompt template with your niche filled in. Ask for the output as a single HTML file with embedded CSS and JS. No external dependencies.

KEY REQUIREMENTS TO ADD TO EVERY PROMPT: ✓ "Output as single HTML file, no external libs" ✓ "Mobile-first responsive design" ✓ "Add comments: " ✓ "Make it feel fast and interactive" ✓ "Include meta tags for SEO"
⚡ Speed TipIterate fast. If the first output is 80% good, ask Claude to fix specific parts. Don't start over. You can have a complete site in 3–5 messages.
03
HTML Setup & Testing
Getting your file ready for the web
✓
▼

Copy the generated HTML from Claude. Open a text editor (VS Code is free and ideal). Create a new file called index.html. Paste the code.

Test locally by double-clicking the file — it will open in your browser. Check mobile by resizing your browser window. Fix anything that looks off by going back to Claude with specific instructions.

FILE STRUCTURE (keep it simple): /your-site-name/ └── index.html ← everything in here └── (optional) /images/
🔍 SEO Quick WinsMake sure your index.html has a descriptive <title> tag and meta description. These take 2 minutes and matter a lot for free organic traffic.
04
GitHub Hosting (Free)
Live on the internet in under 5 minutes
✓
▼

GitHub Pages gives you free hosting forever. No credit card. No expiry. Your site gets a free domain: username.github.io/sitename. You can also connect a custom domain later.

STEPS: 1. Create free account at github.com 2. Click "New Repository" 3. Name it anything (e.g. iq-test) 4. Upload your index.html file 5. Go to Settings → Pages 6. Set Source: "main branch" 7. Wait 60 seconds → your site is live ✓
🌐 Custom Domain (optional) Buy a .com for ~$10/year on Namecheap. Point it to GitHub Pages. Now you look professional and ads pay better rates on real domains vs github.io subdomains.
05
Ad Integration & Revenue
Turning traffic into money
✓
▼

The fastest path to revenue: Google AdSense for beginner sites. Once you have consistent traffic, upgrade to Ezoic, Adsterra or Mediavine for 3–5x higher RPM.

AD PLACEMENT STRATEGY: • Before results → highest engagement moment • After form submit → user is waiting anyway • Sidebar → passive impressions • Footer → captures scroll-throughs AVOID: Ads above the fold on mobile (violates policies + kills user experience)
💵 RPM Reality CheckStart at $2–5 RPM. Entertainment niches. As you grow and switch networks, expect $8–20+ RPM. 10,000 monthly visits × $10 RPM = $100/mo per site. Stack 10 sites = $1,000/mo passive.
